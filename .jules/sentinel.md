## 2025-02-14 - Prevent Basic Auth Transmission Over HTTP
**Vulnerability:** The `init_langfuse_telemetry` function in `hyperinfer-client` accepted any URL scheme for the Langfuse host. Because it uses HTTP Basic Authentication, passing an `http://` URL would transmit the credentials (which act as an API key/password) in Base64 encoding over plaintext, allowing network interception.
**Learning:** Even if the default host is `https://cloud.langfuse.com`, user-provided or configurable URLs must be validated when sensitive credentials are involved. The assumption that users will correctly configure HTTPS for external hosts can lead to severe credential leaks in production environments.
**Prevention:** Always enforce HTTPS for outbound requests carrying sensitive data (like Basic Auth, Bearer tokens, or API keys). Make explicit, hardcoded exceptions for local development environments (`localhost`, `127.0.0.1`) if necessary, but fail fast and hard if an insecure scheme is used for a remote host.
